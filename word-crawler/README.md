# 香港小學學習字詞表 抓取工具

自動從教育局《香港小學學習字詞表》網上查詢系統
（<https://www.edbchinese.hk/lexlist_ch/>）抓取**全部**學習字詞，並匯出 CSV。

## 背景

原 PDF（`requirement.md` 所附連結）只有一頁介紹，真正的資料在教育局的網上檢索系統。
該系統背後的資料庫其實是《常用字字形表》共 **4,762 字**，其中屬《香港小學學習字詞表》
的字，本次抓取實得 **3,129 字**（判別方法見下）。判別方法：凡屬字詞表的字，其結果頁
都會出現「小學學習字詞表」表格（HTML `id="tblCi"`），並以 `ks1` / `ks2` 標示第一 /
第二學習階段；不屬字詞表的字（例如「宋」）只有《常用字字形表》附表，沒有學習階段。
本工具即以查詢系統實際回傳的 `tblCi` 表為準，逐字判定，不預設固定字數。

> 註：坊間常引用「3,171 字」等數字，源自官方文件的概述，與線上查詢系統即時回傳的
> 結果略有出入。本工具的所有統計（字數、階段分佈、詞語數）皆由實際抓取的頁面即時
> 計算，以查詢系統為唯一依據，不採用外部估算值。

## 抓取流程

1. 以「總筆畫數」1–32 逐一查詢 `charlist.jsp`，列舉資料庫全部字的 `id`（去重）。
2. 逐一抓取 `result.jsp?id=XXXX` 詳細頁，用 BeautifulSoup 解析所需欄位。
3. 只保留屬《香港小學學習字詞表》的字，匯出 CSV。

原始 HTML 會快取到 `./cache/`，方便**斷點續抓**與重複解析，亦避免對教育局伺服器造成
負擔。中途中斷後再執行，已抓取的頁面會直接讀快取，不會重複下載。

## 需求

- [uv](https://docs.astral.sh/uv/)（會自動安裝 `requests`、`beautifulsoup4`、`lxml`、
  `truststore`，無需手動建立虛擬環境）。

> 註：教育局伺服器提供的 TLS 憑證鏈需以**作業系統信任庫**驗證，故本工具用 `truststore`
> 令 Python 沿用系統憑證（與 `curl`／瀏覽器一致）。若仍失敗可加 `--insecure`。

## 用法

```bash
# 完整抓取，輸出到 ./out/
uv run crawl_lexlist.py

# 測試：只抓前 50 個字
uv run crawl_lexlist.py --limit 50

# 調整並發與延遲
uv run crawl_lexlist.py --workers 8 --delay 0.1

# 連非字詞表的字（僅《常用字字形表》）一併匯出
uv run crawl_lexlist.py --keep-non-list
```

若你的環境（如公司網絡）令 uv 無法連 PyPI 取得依賴，可加 `--native-tls`：
`uv run --native-tls crawl_lexlist.py`。

### 參數

| 參數 | 預設 | 說明 |
| --- | --- | --- |
| `--out` | `out` | CSV 輸出目錄 |
| `--cache-dir` | `cache` | HTML 快取目錄 |
| `--workers` | `6` | 並發下載數 |
| `--delay` | `0.15` | 每次網絡請求後延遲秒數（讀快取時不生效）|
| `--limit` | `0` | 只處理前 N 個字（測試用，0＝全部）|
| `--keep-non-list` | 關 | 連非字詞表的字一併匯出 |
| `--insecure` | 關 | 停用 TLS 憑證驗證（最後手段）|

## 輸出

### `out/characters.csv`（每字一列）

| 欄位 | 說明 |
| --- | --- |
| `id` | 系統內部編號 |
| `繁體字` | 繁體字 |
| `部首` | 部首（已去「部」字，如「一」）|
| `筆畫` | 總筆畫數 |
| `普通話拼音` | 普通話拼音；多讀音以「；」分隔 |
| `粵拼` | 粵語拼音（粵拼）；多讀音以「；」分隔，〔〕內為同音字提示 |
| `學習階段` | 第一學習階段 / 第二學習階段 |
| `字例` | 該字的用例（如「①上山 ②上月」），部分字沒有 |
| `相關詞語數目` | 相關詞語數量 |
| `相關詞語` | 全部相關詞語，以「、」分隔 |

### `out/words.csv`（每個相關詞語一列，長格式）

| 欄位 | 說明 |
| --- | --- |
| `字id` | 對應 `characters.csv` 的 `id` |
| `繁體字` | 所屬單字 |
| `相關詞語` | 詞語 |
| `普通話拼音` | 該詞普通話拼音 |
| `粵拼` | 該詞粵拼 |
| `學習階段` | 該詞所屬學習階段 |

兩個 CSV 皆以 `UTF-8-BOM` 編碼，可直接用 Excel 開啟而不亂碼。

## 在 GitHub 上瀏覽（Markdown）

CSV 適合下載／匯入，但不便直接在網頁翻閱。故另附一個小工具把 CSV 轉成 GitHub
可直接渲染的 Markdown 表格（按總筆畫分節、附目錄）：

此瀏覽用的 Markdown 位於版本庫根目錄的 `香港小學學習字詞表/`（與 `word-crawler/`
同層）。於版本庫根目錄執行：

```bash
uv run 香港小學學習字詞表/to_markdown.py   # 讀 word-crawler/out/*.csv，生成下列 Markdown
```

- **[`../香港小學學習字詞表/characters.md`](../香港小學學習字詞表/characters.md)** — 全部
  3,129 字，按筆畫分節，檔首附目錄可跳轉；每列含部首、拼音、粵拼、階段、詞數及該字
  全部相關詞語。
- **[`../香港小學學習字詞表/words/`](../香港小學學習字詞表/words/)** — 相關詞語（連逐詞
  普通話拼音／粵拼／階段），按字的筆畫分檔（`stroke-01.md` … 每檔皆小，渲染穩定），
  另有 `README.md` 目錄。

> 為何拆檔：GitHub 對過大的檔案會改顯示原始碼而不渲染。故 `words` 依筆畫拆成多個
> 小檔（最大約 90 KB），並避免把 17,704 列塞進單一表格。

## 實際結果

以下為本工具實際抓取、由 CSV 直接統計所得（非外部估算）：

- 資料庫（常用字字形表）：**4,762** 字
- 《香港小學學習字詞表》：**3,129** 字
  - 第一學習階段：**1,921** 字
  - 第二學習階段：**1,208** 字
  - 其中 **239** 字沒有相關詞語（只有單字條目）
- 相關詞語：**17,704** 條（`words.csv` 長格式，每字每詞一列）
  - 去重後不同詞語約 **8,317** 條（同一詞語常見於多個組成字之下，例如「三文治」會
    同時出現在「三」「文」「治」之下）

> 字級欄位（部首、筆畫、普通話拼音、粵拼、學習階段）在全部 3,129 字皆完整無缺。
> 個別詞語（如「水庫」）在查詢系統中同時列於 `ks1` 與 `ks2`，本工具照實保留，
> 不作合併。
